shadeyg56:

Both programs essentially do the same thing cpufreq works by allowing the user to manually control their CPU frequencies, turbo, etc... The idea behind auto-cpufreq is that the user doesn't have to manually control this stuff, so the program will do it for you, hence the **auto** in the name. In this case, you should choose which program you prefer and remove the other one. If you are interested in the GUI of cpufreq, you might be interested to know that this is being worked on in PR #486
