[Request] Separate the package management part from `core.py`, use an independent installer and the correct location (`/usr/local/bin`)

Issue #449

Description

According to [file-hierarchy(7)](https://man.archlinux.org/man/file-hierarchy.7.en), `/usr` (except for `/usr/local`) should contain _Vendor-supplied operating system resources_ i.e. shall be managed by the package manager and the package manager only. Runtime-generated files like `cpufreqctl.auto-cpufreq` should be installed to `/usr/local/bin`. https://github.com/AdnanHodzic/auto-cpufreq/blob/master/auto_cpufreq/core.py#L309
