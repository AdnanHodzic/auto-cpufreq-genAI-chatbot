Hello, thank you for the feature request and provided info. how auto-cpufreqs battery management works is uses kernels modules ( code for linux that is added on demand to allow linux to access certain things eg setting battery charge thresholds ) different laptops require different kernel modules to access battery functions. ive done some research [arch wiki](https://wiki.archlinux.org/title/Laptop/HP) and [tlp docs](https://linrunner.de/tlp/settings/bc-vendors.html) and unfortunately i haven't found any evidence leading to hp having a kernel module. however you should try installing [TLP](https://github.com/linrunner/TLP) and see if that has support if it does reply back with tlp-stat output.
