Installation package clobbers system-wide `cpufreqctl`

Issue #149

Description

The installation process in both snap and native package managers moves the existing `cpufreqctl` script out of the way to replace it with the one bundled with `auto-cpufreq`. This violates the integrity of the `cpufreqctl` package, because the system is no longer running the version that the package manager installed. Two options that can solve this issue: * depend on the `cpufreqctl` package in the various distributions * place the bundled version so that it doesn't clobber the package manager provided `cpufreqctl` I've opted for the second approach as this has the least impact, and suggest that the bundled version of this file is placed beside the one provided by the package manager, as say `cpufreqctl.auto-cpufreq`. This way the system integrity is maintained and `auto-cpufreq` can still rely on its own (and potentially modified) version of this script.
