NoProsNoNoobs:

I have found the solution a while back, but kinda forgot about this post. If you want auto-cpufreq to run live, you need to first run the command `auto-cpufreq --install`. This will install the daemon it could not find in my error. After that, you can run `auto-cpufreq --live` without problems and it will save the changes. If you want to remove the daemon again you can run `auto-cpufreq --remove`. Auto-cpufreq works really great now and extends my laptop's battery life by about 2 to 3 hours.
