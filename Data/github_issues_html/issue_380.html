0x0013:

I think this issue is meant to request similar feature as #315 and #356 . Currently, via the config file, it is possible to configure 2 different profiles based on the dimension of current power state (charger/battery). What I see from these requests are similar features but for slightly different use-cases: #315 use-case (I'm assuming) - powersave governor when discharging and on low battery, but more power when battery is still high. #356 use-case - powersave governor while charging on low battery, for faster battery level replenishment this issue use case (I'm assuming) - change other parameters (like turbo boost) based on battery level What I think would unify this and the above mentioned feature requests, would be to add a second dimension to the selection of battery level as the configured profiles. Currently configurable profiles are already configurable enough (governor, turbo boost, min/max frequercy). If battery level dimension could be added to the selection criteria, I think it would satisfy these requests. Ideally, both dimensions would be optional - some people want specific settings, but only based on charging/plugged status (as it is now), while others might want only based on battery level (i.e. powersave below 50%, performance above, don't care if charging or discharging), while others yet might want both (powersave only under 50%, but always performance when plugged-in). Though this does complicate things a little, as then it's needed to take into account which selection takes priority if two single-dimension states are true (i.e. profile exists for "below 40%" and profile exists for "plugged-in", and laptop is 30% while charging)
