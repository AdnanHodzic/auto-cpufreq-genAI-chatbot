psutil 7.0.0

Issue #817

Description

psutil version 7.0.0 was released [upstream](https://pypi.org/project/psutil/#history). I saw this project requests any 6.x version. The [release notes](https://github.com/giampaolo/psutil/blob/master/HISTORY.rst#700) only list incompatibilities with python2.7 (this project already [uses newer py](https://github.com/AdnanHodzic/auto-cpufreq/blob/f2c4a2d763551e64c82469c52767762d01a4d994/pyproject.toml#L24)), and removal of `memory_info_ex` which this project does not use at all. This means 7.0.0 should "just work". Any chance to unpin this in the `pyproject.toml`? Currently, attempting to build against 7.0.0 obviously fails (which i noticed in the [nixpkgs ci](https://hydra.nixos.org/build/292453144/log)). Figured i'd open an issue, partially to track for myself. I understand this is primarily a packaging thing and failing on the nix side, but by the looks of it supporting the new release of psutil shouldn't be too hard.
