Why display temperature per THREAD, why not per CORE?

Discussion #885

Description

Hi, On AMD CPU (at least) auto-cpufreq GUI shows separate temperature for each (hyper)THREAD, i.e. **DOUBLE** the number of cores. I doubt the temperatures between 2 hyperthreads of the **same core** may be different, so why not display one same temperature per core? Or even add an **option** to show average temperature only? (not the main topic here). There's already quite a lot of info in the GUI, and number of cores is quickly growing recently. Also requesting temperature twice per each core clearly doesn't improve the auto-cpufreq performance. Thank you.
