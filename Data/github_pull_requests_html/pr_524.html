[BUG-FIX] Adding root_check before set_override method execution

Pull Request #524

Description

Fixes permission issue prompt when calling `--force` flag in non root mode. This PR, basically calls pre-defined `root_check()` function before accessing `set_override` method aimed at implementing the force feature. And hence ensuring `set_override` calls are always in su mode. So, in effect all override using 'force' can clearly access the `governer_override_state` path file, and write changes to it, without externally tinkering the environment.
