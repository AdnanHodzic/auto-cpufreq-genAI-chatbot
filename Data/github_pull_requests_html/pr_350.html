Stop power-profiles-daemon in live mode

Pull Request #350

Description

When using `auto-cpufreq --live` we did only display a warning about the conflict with `power-profiles-daemon`. It now actually stops it in live mode, and starts it when live mode is exited. Also added a check to see if `powerprofilesctl` exists.
